Especificación de los componentes de JEE

Enterprise JavaBeans (EJB)
Los EJB proporcionan un modelo de componentes distribuido estándar del lado del servidor. El objetivo de los EJB es dotar al programador de un modelo que le permita abstraerse de los problemas generales de una aplicación empresarial (concurrencia, transacciones, persistencia, seguridad, etc.) para centrarse en el desarrollo de la lógica de negocio en sí. El hecho de estar basado en componentes permite que Seacute/stos sean flexibles y sobre todo reutilizables.

Servlets
Los servlets son objetos que corren dentro y fuera del contexto de un contenerdor de servlets (ej: Tmocat) y extienden su funcionalidad. La palabra servlet deriva de otra anterior, applet, que se refería a unos pequeños programas que se ejecutan en el contexto de navegador web, por contraposición. El uso más comun de los servlets es generar paginas web de forma dinámica a partir de los parámetros de la petición que envía el navegador web.
Servlets
Los portlets son componentes modulares de las interfaces de usuario gestionadas y visualizadas en un portal web. Los portlets producen fragmentos de codigo de marcado que se agregan a la pagina de un portal. Típicamente, siguiendo la metáfora de un escritorio; la pagina de un portal se visualiza como la colección de ventanas de un portlet que no se solapan; dónde cada una de estas muestra un portlet. Por lo tanto, un portlet se semeja a una aplicacion web que esta hospedade en un portal. Como por ejemplo, un portlet de aplicacion puede ser para el correo, el parte meteorológico, un foro, noticias, etc.
Java server pages
JavaServer Pages (JSP), es una tecnologia que ayuda a los desarrolladores de software crear paginas web donamicas basadas en HTML, XML entre otros tipos de documentos, JSP es similar a PGP, perp utiliza el lenguaje de programacion Java.
